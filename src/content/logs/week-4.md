### Overview
Week 4 (March 2 – March 8) was a high-velocity development phase focused on the Hack4Mapandan initiative. The goal was to transform research into a functional prototype for the Municipal Disaster Risk Reduction and Management Office (MDRRMO), focusing on emergency verification and administrative oversight.

### SVS Development & Design
Concept & Presentation
Project Pitch: Created a comprehensive web presentation for the Disaster Response & Emergency Verification system.

Workflow Visualization: Finalized a detailed PowerPoint presentation explaining the emergency report-to-dispatch lifecycle.

Municipal Alignment: Tailored the system features to address specific communication gaps identified during the research phase.

Branding & Identity
Logo Refinement: Improved and finalized the official logo for the system.

Visual Trust: Focused on a professional, high-visibility design that conveys safety and municipal authority.

### Technical Implementation
Admin Dashboard
Backend Integration: Successfully added the Admin Dashboard to manage incoming reports.

Stability & Optimization: Conducted a "bug-bash" session to ensure technical stability, focusing on database reliability and report routing.

Reliability: Implemented fixes for critical workflow bugs to ensure the system remains operational during high-stress scenarios.

### Tech Stack
Styling: Tailwind CSS

Backend: Node.js & EJS

Presentation: Microsoft PowerPoint & Google Slides

Version Control: GitHub

### Key Contributions
System Architecture:

Bridged the gap between public reporting and administrative verification.

Established a secure login flow for municipal administrators.

Communication Design:

Produced high-quality presentation materials to explain technical workflows to non-technical stakeholders.

Refined the brand identity to ensure municipal verification is easily recognizable.

### Lessons Learned
Presentation is Key: Being able to visualize the workflow via a pitch deck is just as important as the code when working with municipal clients.

Stability Over Features: In disaster management, a stable system with fewer features is better than a complex system that crashes during an emergency.

Iterative UI: Improving the logo and layout based on daily feedback helps build immediate trust with users.
